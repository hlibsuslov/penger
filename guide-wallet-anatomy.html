Anatomy of a Wallet

How cryptocurrency wallets work under the hood — from key derivation to transaction signing.

What Is a Crypto Wallet?

A cryptocurrency wallet is a tool that stores your private keys and allows you to send, receive, and manage digital assets. Despite the name, wallets don't actually store your coins — those live on the blockchain. Your wallet holds the keys that prove ownership.

There are many wallet forms, each with different trade-offs:

  • Mobile apps — convenient, always with you
  • Desktop software — more control, larger screen
  • Browser extensions — seamless DeFi access
  • Hardware devices — maximum security, keys never online

Public Keys vs. Private Keys

Every wallet has at least one key pair:

  • Public key — used to receive funds (like an email address)
  • Private key — used to authorize spending (like a password)

Anyone can send crypto to your public address, but only the private key holder can move it out.

Modern HD wallets generate thousands of key pairs from a single seed phrase. One backup protects access to all your addresses across multiple cryptocurrencies.

Addresses and Receiving Funds

An address is derived from your public key through hashing and encoding. For privacy, best practice is to use a new address for each transaction.

Best Practice HD wallets handle address generation automatically by producing sequential addresses from the same seed. Never reuse addresses.

BIP32 Hierarchical Deterministic Derivation

BIP32 defines a tree structure for key derivation. Starting from a master key (derived from the BIP39 seed via PBKDF2), child keys are generated using HMAC-SHA512.

Each level in the tree can produce 2^31 normal child keys and 2^31 hardened child keys.

Hardened derivation (indicated by an apostrophe, e.g., m/44') prevents child key compromise from leaking information about the parent key. This is critical for multi-account security isolation.

ECDSA and Transaction Signing

Bitcoin uses the secp256k1 elliptic curve for digital signatures. When you send a transaction, your wallet:

  1. Creates a hash of the transaction data
  2. Signs it using your private key
  3. Broadcasts the signature with the transaction
  4. Network nodes verify against your public key

The fundamental cryptographic primitive is elliptic curve multiplication — computationally easy in one direction but infeasible to reverse.

Extended Public Keys (xpub) and Watch-Only

An extended public key (xpub) allows deriving all public child keys without access to the private key. This enables:

  • Watch-only wallets — monitor balances without spending ability
  • Payment servers — generate fresh addresses for each customer
  • Auditing — third-party balance verification
Security Note Never share your xpub publicly. While it can't spend funds, it reveals your entire transaction history and all addresses.
