Anatomy of a Seed Phrase

What Is a Seed Phrase?

A seed phrase is a set of 12 to 24 words that serves as the master backup for your cryptocurrency wallet. These words are selected from a standardized list of 2,048 English words defined by the BIP39 specification.

Think of a seed phrase like the master key to a building. With this single key, you can regenerate every room key (private key) and access every lock (address) in the building.

Losing this key means losing access to everything inside — permanently. There is no "forgot password" option in self-custody crypto.

How Are the Words Chosen?

Your wallet uses a cryptographically secure random number generator (CSPRNG) to produce a sequence of random bits called entropy. This entropy is then processed through a series of steps:

  1. Random entropy is generated (128 bits for 12 words)
  2. A SHA-256 checksum is calculated and appended
  3. The combined bits are split into 11-bit segments
  4. Each segment maps to a word from the BIP39 wordlist

Each word corresponds to a specific number between 1 and 2048. The mapping is deterministic — the same number always produces the same word.

Why Is Order Important?

The order of your seed words matters. Changing the position of even one word creates an entirely different wallet with different keys and addresses.

Important Always write your words down in the exact order they appear. Number each word (1, 2, 3…) to avoid confusion when restoring your wallet later.

A 12-word phrase has 2048^12 possible combinations — roughly 5.4 × 1039. Guessing or brute-forcing the correct sequence is computationally infeasible.

Entropy Sources and CSPRNG

BIP39 seed generation begins with a cryptographically secure pseudo-random number generator (CSPRNG). On modern systems, this is typically backed by:

  • /dev/urandom on Linux
  • CryptGenRandom on Windows
  • The Web Crypto API in browsers

The entropy length determines the word count: 128 bits produces 12 words, 160 bits produces 15 words, and so on up to 256 bits for 24 words. Each additional 32 bits of entropy adds 3 words and one checksum bit.

SHA-256 Checksum Mechanism

After generating raw entropy, the system computes a SHA-256 hash of the entropy bytes. The first N bits of this hash (where N = entropy_bits / 32) are appended to the entropy as a checksum.

The checksum serves as an error-detection mechanism. If you mistype or misspell a word during recovery, the checksum will likely fail, alerting you to the error.

This combined bit string is then divided into 11-bit segments, each mapping to one of the 2,048 BIP39 words.

From Mnemonic to Master Key (BIP32/BIP44)

The mnemonic phrase is converted to a 512-bit seed using PBKDF2-HMAC-SHA512 with 2,048 iterations. The passphrase "mnemonic" + optional user passphrase is used as the salt.

Derivation Paths

This seed feeds into BIP32 hierarchical deterministic key derivation. The path determines which specific key pair is generated:

  • m/44'/0'/0'/0/0 — Bitcoin (BIP44)
  • m/44'/60'/0'/0/0 — Ethereum
  • m/84'/0'/0'/0/0 — Bitcoin SegWit (BIP84)

Each path segment narrows down from the master key to a specific account and address index.
